# Reproduce steps for a simple shim DLL

## Create IQ-TREE static library

### Create simple function to export

In `iqtree/main/` create `bridge_fun.h` and `bridge_fun.cpp` with the following contents:

`bridge_fun.h`

```cpp
#ifndef BRIDGE_FUN_H
#define BRIDGE_FUN_H

#include <string>

using namespace std;

extern "C" int bridge_rf(string& tree1, string& tree2);

#endif
```

`bridge_fun.cpp`

```cpp
#include "bridge_fun.h"

extern "C" int bridge_rf(string& tree1, string& tree2) {
  return 42;
}
```

### Include files in build process

In `iqtree2/main/CMakeLists.txt` add `bridge_fun.h` and `bridge_fun.cpp` to the main library when building lib.

### Build IQ-TREE static library

Create in the `iqtree2` directory create a new folder called build. Then in that directory run the following two commands in sequence

`cmake -G "MinGW Makefiles" -DCMAKE_C_COMPILER=clang -DCMAKE_CXX_COMPILER=clang++ -DCMAKE_MAKE_PROGRAM=make -DCMAKE_C_FLAGS="-IC:/ProgramData/chocolatey/lib/eigen/include/eigen3 --target=x86_64-pc-windows-gnu" -DCMAKE_CXX_FLAGS="-IC:/ProgramData/chocolatey/lib/eigen/include/eigen3 --target=x86_64-pc-windows-gnu" -DBUILD_LIB=ON ..`

`make -j`

## Create a Shim DLL

### Setup Folder

Create a new folder called `bridge` and copy `iqtree2/build/main/libmain.a`
generated by the previous step into that folder. (I could've sworn I got this to work through `libiqtree2.a` before, but apparently not so now).

### Check the static library works

In the `bridge` folder create two new files

`bridge.h`

```cpp
#ifndef BRIDGE_FUN_H
#define BRIDGE_FUN_H

#include <iostream>
#include <string>

using namespace std;

extern "C" int bridge_rf(string& tree1, string& tree2);

#endif
```

`bridge.cpp`

```cpp
#include "bridge.h"

int main() {
  string tree1 = "(a,b,(c,(d,e)));";
  string tree2 = "(e,b,(c,(d,a)));";

  cout << bridge_rf(tree1, tree2) << endl;
  return 0;
}
```

Compile the program with:

```bash
g++ bridge.cpp -lmain -L.
```

The executable should print 42 to the terminal.

### Create MSVC compatible dynamic library

Create a dll and def file with the following

```bash
g++ -shared -o bridge.dll bridge.cpp -Wl,--output-def,bridge.def -lmain -L.
```

### Use MSVC's lib.exe to degnerate lib file

```bash
lib /def:bridge.def
```

This generates a .lib file which links to the dll, which I have verified can link to piqtree.
